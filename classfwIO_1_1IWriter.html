Detailed Description

Writer service API. It manages extension points definition and extension configuration.

Slots

  • setFileFolder(const ::boost::filesystem::path&) : Sets the folder when a path is configured in FILE or FILES mode

This class represents the base interface for writer services. Use the base service methods :

  • The service is configured with methods setConfiguration(cfg) and configure()
  • The method start() initialize the service
  • To write the object use update() method
  • Finally we must call stop() before deleting the service

void fwIO::IWriter::configuring ( )
overrideprotectedvirtual

This method proposes to parse xml configuration to retrieve file/files/folder paths.

You can implement your configuring method if you wan check another information or information not correspond to a path on filesystem, else you must use this generic approach.

Sample configuration for a file:

1 <service ... >
2  <file>/home/user/myFile.jpg<file/>
3 </service>

Sample configuration for many files:

1 <service ... >
2  <file>/home/user/myFile01.jpg<file/>
3  <file>/home/user/myFile02.jpg<file/>
4  <file>/home/user/myFile03.jpg<file/>
5 </service>

Sample configuration for a folder:

1 <service ... >
2  <folder>/home/user/myFolder<folder/>
3 </service>

You may specify the title of the modal file selection window with the windowTitle config attribute as such:

1 <service ... >
2  <windowTitle>Example window title</windowTitle>
3 </service>

Implements fwServices::IService.

fwIO::IOPathType fwIO::IWriter::getIOPathType ( ) const
virtual

This method must be implemented by concrete service writers that use path file system to write data.

This method returns supported path format (file, files or folder). A reader can support file and folder, or files and folder, but not file and files ( because files include file concept ).
