Detailed Description

WindowLevel service allows to change the min / max value of windowing.

This is represented by two sliders to modify the min and max values of windowing

Slots

  • updateTFPoints() : update the slider according to the new points
  • updateTFWindowing(double window, double level) : update the slider according to the new window and level

XML Configuration

1 <service uid="..." type="::uiImageQt::WindowLevel" autoConnect="yes">
2  <inout key="image" uid="..."/>
3  <inout key="tf" uid="..." optional="yes" />
4  <config autoWindowing="yes" enableSquareTF="no" />
5 </service>

In-Out

  • image [fwData::Image]: image on which the windowing will be changed
  • tf [fwData::TransferFunction] (optional): the current TransferFunction. If it is not defined, we use the image's default transferFunction (CT-GreyLevel). The transferFunction's signals are automatically connected to the slots 'updateTFPoints' and 'updateTFWindowing'.

Configuration

  • autoWindowing(optional, default no) : if 'yes', image windowing will be automatically compute from image pixel min/max intensity when this service receive BUFFER event
  • enableSquareTF(optional, default: yes) : if 'yes' enable the button to switch between current TF and square TF
