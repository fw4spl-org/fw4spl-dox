Detailed Description

An updater is attached on a composite. It is use to add, swap and remove object in the composite when it receives specific message (declared in configuration) and sends corresponding message message. It work mainly with a IManager.

void ctrlSelection::IUpdaterSrv::configureManagedEvents ( ::fwRuntime::ConfigurationElement::sptr  configuration) [protected]

Configure the event to manage and its corresponding action.

Parameters:
configurationconfiguration of the service

Sample of declaration configuration :

         <update compositeKey="reconstructionKey" onEvent="NEW_RECONSTRUCTION" fromUID="modelSeriesUid" actionType="ADD_OR_SWAP" />
         <update compositeKey="reconstructionKey" onEvent="CLEAR" fromUID="modelSeriesUid" actionType="REMOVE" />

Here, when the updater receives a message with the NEW_RECONSTRUCTION event from the object "modelSeriesUid" the object reconstructionKey is added or swapped in the composite. When the updater receives a message with the CLEAR event from the object "modelSeriesUid" the object reconstructionKey is removed from the composite

The actionType can be : ADD, ADD_OR_SWAP, SWAP, REMOVE, REMOVE_IF_PRESENT and DO_NOTHING
